# Lab 9 DDR RTL

This directory extends `lab_8_interrupt` with a wait-state data memory path and a DDR2 backend for Nexys4 DDR.

## Main modules

- `top.v` — hardware top for Nexys4 DDR. It instantiates the 100->200 MHz clock generator, true MIG IP wrapper (`lab9_mig` generated by Vivado), the SoC, CDC, and AXI adapter.
- `soc.v` — MIPS SoC wrapper. The CPU runs on `clk`; `simple_bus` runs on `bus_clk` to preserve the half-cycle bus timing used by earlier labs.
- `simple_bus.v` — address decoder for boot ROM, BRAM, MMIO, DDR status, and the DDR data window.
- `ddr_bridge.v` — CPU-side blocking, single-outstanding DDR request/response bridge.
- `ddr_backend_cdc.v` — toggle-based CDC between the 50 MHz teaching SoC bus domain and the MIG UI clock domain.
- `mig_axi_adapter.v` — maps 32-bit bridge requests into 64-bit single-beat AXI transactions for MIG.
- `ddr_model.v` — fast behavioral DDR-like model used only in simulation.
- `clk_100_to_200.v` — MMCM wrapper that generates the MIG system clock from the board 100 MHz clock.
- `mig/nexys4ddr_mig.prj` — Digilent Nexys4 DDR C.1 MIG project file used by `scripts/build_lab9.tcl` to generate the MIG IP.
- `mips_core/` — lab9 copy of the MIPS pipeline with `d_validM/d_readyM` wait-state support.

## Address highlights

- DDR2 data window: `0x8000_0000 - 0x87ff_ffff`
- DDR status MMIO: `0x1000_4000`
  - bit0: calibration done
  - bit1: busy

## Simulation vs hardware

Simulation excludes the physical MIG IP and connects `soc` to `ddr_model`. Hardware build excludes `ddr_model` and generates `lab9_mig` from `mig/nexys4ddr_mig.prj`.

## Expected hardware output

At `115200 8N1`, the boot ROM prints a banner, waits for DDR calibration, runs pattern tests in DDR, and then repeatedly prints:

```text
DDR PASS
```

The repeated PASS output makes board bring-up robust even if the USB-UART device re-enumerates during JTAG programming.
